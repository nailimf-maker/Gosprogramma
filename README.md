# Дашборд расселения аварийного жилья

Публичный аналитический дашборд программы расселения аварийного жилищного фонда
(Московская область). Статический сайт, разворачивается на GitHub Pages.
33 183 записи реестра выгружаются в один компактный `data.json` (~1,4 МБ; ~155 КБ
под gzip, который GitHub Pages включает автоматически). Все срезы и любые комбинации
фильтров считаются прямо в браузере — без сервера и API.

## Что показывает

- Прогресс расселения по региону — лента статусов + ключевые показатели.
- Рейтинг 49 муниципальных районов с фильтрацией одним кликом.
- Срезы: состояние расселения, тип собственности, тип помещения, комнатность.
- Динамика признания домов аварийными и накопительное завершение расселения.
- Топ‑40 домов с наибольшим объёмом нерасселённых помещений.

### Фильтры (комбинируются между собой)

- **Район** — выпадающий список или клик по строке рейтинга.
- **Причина аварийности** — физический износ / пожар / влияние окружающей среды.
- **Признан аварийным** — диапазон лет (с–по), 1997–2026.

Любая комбинация фильтров мгновенно пересобирает все показатели и графики:
данные считаются прямо в браузере по «фактам», без обращений к серверу.

## Структура

```
.
├── index.html              дашборд (вёрстка, стили, логика)
├── data.json               «факты» в колоночном виде (артефакт сборки)
├── build_data.py           генератор data.json из исходного xlsx
├── vendor/echarts.min.js   библиотека графиков (вендорится локально, без CDN)
├── .nojekyll               отключает обработку Jekyll на Pages
└── .github/workflows/      автодеплой на GitHub Pages
```

ID‑колонки реестра (`ID СФ`, `ID МР`, `ID МО дома`, `ID Дома`, `ID помещения`)
в дашборд не выводятся — отбрасываются на этапе сборки.

## Обновление данных

При новом отчётном периоде нужно пересобрать только `data.json` — `index.html`
не трогается.

```bash
python3 -m pip install pandas openpyxl
python3 build_data.py "Новая_версия2.xlsx" data.json
git add data.json && git commit -m "data: обновление за <период>" && git push
```

Исходный xlsx в репозиторий не коммитится (персональные данные о жильцах).
`data.json` содержит только агрегаты без адресной привязки к людям.

## Локальный просмотр

Дашборд использует `fetch('data.json')`, поэтому открывать файл напрямую
(`file://`) нельзя — нужен http-сервер:

```bash
python3 -m http.server 8080
# открыть http://localhost:8080
```

## Деплой на GitHub Pages

1. Создать публичный репозиторий и запушить содержимое этой папки в ветку `main`.
2. Settings → Pages → Source: **GitHub Actions**.
3. Workflow `deploy.yml` соберёт и опубликует сайт автоматически при каждом пуше
   в `main`. Адрес появится в логе деплоя.

## Технические характеристики

- Зависимости: Apache ECharts (вендорится локально), Hanken Grotesk + IBM Plex Mono (Google Fonts).
- Доступность: семантическая разметка, видимый фокус с клавиатуры, поддержка `prefers-reduced-motion`.
- Адаптивность: от десктопа до мобильного (≤ 680 px).
- Без сборщиков и runtime-зависимостей — чистый статический сайт.
